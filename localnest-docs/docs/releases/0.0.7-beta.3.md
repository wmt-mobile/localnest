## Summary

`0.0.7-beta.3` is a reliability and DX release. It fixes KG validation errors that blocked nullable fields, improves the skill installer to be more resilient, removes redundant schema indexes, and introduces `nest:*` agentic orchestration commands for structured multi-agent workflows.

## Fixes

### Knowledge Graph Validation
- `kg_add_triple` and `kg_invalidate` now accept `null` for `valid_from`, `valid_to`, and `source_memory_id`. Previously these calls were rejected with a validation error when these optional fields were omitted or explicitly set to null.

### Schema
- Removed redundant indexes in `schema.js` that were already created by migration scripts. No functional change; cleaner startup.

### Skill Installer
- Slash commands now sync even when the skill file is already up-to-date. Previously, re-running the installer with no skill changes would skip command registration.
- Claude Code hooks (pre/post tool for memory retrieval and outcome capture) are now auto-installed alongside the skill.
- Removed duplicate log message during skill installation.

## New Features

### `nest:*` Agentic Orchestration Commands

11 new slash commands for structured multi-agent workflows:

| Command | Purpose |
|---------|---------|
| `nest:flow` | Orchestrator -- routes feature requests through the full pipeline |
| `nest:research` | Research agent -- investigate approach, patterns, risks |
| `nest:implement` | Implementation agent -- write code following LocalNest conventions |
| `nest:test` | QA agent -- run tests, lint, verify build, diagnose failures |
| `nest:review` | Code review agent -- security, quality, backwards compatibility |
| `nest:docs` | Docs agent -- maintain Docusaurus site, changelog, skill refs |
| `nest:release` | Release agent -- version bumps, changelog, branch creation |
| `nest:cicd` | CI/CD agent -- GitHub Actions workflows, pipeline health |
| `nest:hotfix` | Hotfix agent -- fast-track bug fixes with minimal overhead |
| `nest:skill-sync` | Skill sync agent -- update skill definitions for all AI clients |
| `nest:help` | Show all available orchestration commands |
