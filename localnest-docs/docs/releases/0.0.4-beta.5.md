## Summary

`0.0.4-beta.5` established the top-level `localnest` command flow for setup, doctor, and upgrade tasks.

## Key Changes

- added top-level `localnest upgrade` as the canonical upgrade path
- updated setup and install guidance to prefer top-level `localnest` commands
- improved upgrade validation and user-facing error reporting
- removed the deprecated `localnest update` alias
- removed the experimental backup sync CLI and Google Drive integration

## Practical Impact

This was the release where the recommended user flow shifted away from mixed `npx` helper commands and toward the direct installed `localnest` CLI.

## When to Use This Page

- debugging a machine still pinned to `0.0.4-beta.5`
- comparing beta.5 upgrade behavior against beta.7
- checking when the top-level `localnest upgrade` flow became the default recommendation
