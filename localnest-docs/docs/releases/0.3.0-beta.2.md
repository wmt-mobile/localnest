## Key Changes

### 1. Modern Interactive CLI
We've refactored the core CLI infrastructure to provide a consistent, premium experience:
- **Centralized ANSI Utility**: Every command now uses a unified styling engine (`ansi.ts`) for colors, badges, and layout.
- **Status Symbols**: Standardized success, error, and info icons across all outputs.
- **Premium Box Layouts**: Structured panels and boxes for help screens and diagnostics.

### 2. Interactive TUI Dashboard
The new `localnest dashboard` command provides a real-time view of your local AI ecosystem:
- **Memory Overview**: Track stored memories and recent changes.
- **Knowledge Graph Stats**: Visualize entity and triple counts.
- **Server Health**: Monitor active connections and resource usage.

### 3. Improved Diagnostics (`localnest doctor`)
The `doctor` command has been completely redesigned to provide actionable, structured feedback:
- **Categorized Checks**: Environment, Dependencies, Config, and Runtime checks.
- **Interactive Solutions**: Clear instructions for resolving configuration issues.
- **Verbose Mode**: Detailed tracing for debugging complex setups.

### 4. Advanced Status Reporting
Long-running operations now feature modern spinners and progress indicators:
- **Refined Spinners**: Powered by `ora`, ensuring smooth feedback during indexing.
- **Dynamic Progress Bars**: Accurate tracking for batch memory storage and KG updates.

### 5. Developer Optimization
- **Full JSON Support**: Added `--json` output to all core commands for CI/CD and automation.
- **Quiet Mode**: Optimized `--quiet` flag for non-TTY environments.
- **Type-Checked API**: Resolved all linting and type errors resulting from the TUI refactor.

## Installing 0.3.0-beta.2

```bash
npm install -g localnest-mcp@beta
localnest setup
localnest doctor
```

## Migration Notes

- **Backward Compatible**: All tool names and existing databases are fully supported.
- **Config Migration**: Automatic migration of legacy configs to the new structured format.
- **Dependency Update**: `ora` updated to `v9.3.0` for improved terminal compatibility.
