## Summary

`0.0.6-beta.1` introduces the first soft-deprecation pass for legacy `localnest-mcp-*` helper commands while preserving the `localnest-mcp` server binary used by MCP clients.

## Key Changes

- added canonical `localnest task-context` and `localnest capture-outcome` commands
- converted `localnest-mcp-setup`, `localnest-mcp-doctor`, `localnest-mcp-upgrade`, `localnest-mcp-install-skill`, `localnest-mcp-task-context`, and `localnest-mcp-capture-outcome` into deprecated compatibility wrappers
- kept `localnest-mcp` unchanged so existing MCP client configs do not need migration
- updated runtime help, README examples, and docs to prefer canonical `localnest ...` commands
- aligned bundled skill metadata and release-prep tooling with the beta package version

## Practical Impact

This beta narrows the human-facing CLI surface without forcing an MCP reconfiguration. Users can switch to the canonical `localnest` command path now, while existing helper aliases continue to work with warnings during the transition.

- testing `localnest-mcp@0.0.6-beta.1`
