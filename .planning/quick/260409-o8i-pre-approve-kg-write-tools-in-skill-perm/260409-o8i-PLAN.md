<objective>
Fix subagent permission blockages during bulk KG / memory building by adding per-command MCP tool pre-approvals to the LocalNest Claude Code skill command files.

Background: in a prior dogfooding session, 3 of 4 background agents got blocked when calling `localnest_kg_add_triple` because it wasn't in the pre-approved `allowed-tools` list for `/localnest:fact`. One hit rate limits before unblocking. This plan scopes each `localnest:*` command's allowlist to exactly the MCP tools it actually invokes in its `<process>` block — writes for write commands, reads for read commands — without blanket-approving destructive tools.

Purpose: unblock serial bulk-building workflows where subagents can't auto-approve tool calls.
Output: 7 command manifests updated with per-command `mcp__localnest__*` allowlists.
</objective>

<background>
Claude Code's skill/command permission model:
- Each command manifest is a Markdown file with YAML frontmatter containing an `allowed-tools:` list.
- For built-in tools, entries are names like `Read`, `Bash`, `Glob`, `Grep`.
- For MCP server tools, entries MUST use the fully-qualified form: `mcp__<server-name>__<tool-name>`.
- The MCP server name for LocalNest is `localnest` (confirmed in src/runtime/version.ts: `SERVER_NAME = 'localnest'`), so the LocalNest KG add-triple tool maps to `mcp__localnest__localnest_kg_add_triple`.
- When a subagent invokes a command with a tool NOT in `allowed-tools`, it triggers a serial permission prompt. In background/bulk workflows this effectively blocks execution and can trigger rate limits.

Canonical LocalNest MCP tool names come from src/mcp/tools/*.ts. Verified names for this plan:
- Writes: localnest_memory_store, localnest_memory_update, localnest_memory_add_relation, localnest_memory_remove_relation, localnest_memory_suggest_relations, localnest_memory_capture_event, localnest_kg_add_entity, localnest_kg_add_triple, localnest_kg_invalidate, localnest_ingest_markdown, localnest_ingest_json, localnest_diary_write
- Reads: localnest_memory_recall, localnest_memory_status, localnest_memory_related, localnest_memory_check_duplicate, localnest_kg_query, localnest_kg_stats, localnest_kg_as_of, localnest_kg_timeline, localnest_graph_traverse, localnest_graph_bridges, localnest_server_status, localnest_health, localnest_hooks_stats, localnest_hooks_list_events, localnest_task_context, localnest_capture_outcome, localnest_search_files, localnest_search_code, localnest_search_hybrid, localnest_read_file, localnest_list_roots, localnest_list_projects, localnest_index_status, localnest_embed_status, localnest_nest_list, localnest_nest_branches, localnest_nest_tree

EXCLUDED FROM ALL ALLOWLISTS (destructive / system-level):
- localnest_memory_delete
- localnest_update_self
- Any other tool with destructive side effects not listed above
</background>
