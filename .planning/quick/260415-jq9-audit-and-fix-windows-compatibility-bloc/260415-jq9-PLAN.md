# Quick Plan 260415-jq9

## Goal

LocalNest is supposed to be cross-OS but Windows users report install/runtime failures. Static audit identifies several critical Windows-hostile patterns that must be fixed for `npm install -g localnest-mcp` to actually work on Windows. Then add a Windows CI matrix so regressions are caught automatically. Then bump version.

## Identified blockers

### CRITICAL — breaks install or basic runtime

| # | File | Bug |
|---|------|-----|
| B1 | `bin/_boot.cjs` | Detects which bin script to launch via `basename(process.argv[1], '.cjs')`. npm doesn't create symlinks on Windows — it generates `.cmd` shims that all spawn `node bin/_boot.cjs ...`, so the basename is always `_boot`. Result: every bin command (`localnest`, `localnest-mcp`, `localnest-mcp-setup`, `localnest-mcp-doctor`, ...) is broken on Windows. |
| B2 | `src/runtime/config.ts:106` | `expandHome()` uses `process.env.HOME` directly (undefined on Windows) and the regex only matches `~/` — never `~\`. Any user config with `~` for paths silently expands to `''`. |
| B3 | `scripts/runtime/preinstall-cleanup.mjs` | (a) line 26 builds Unix root with `path.join('/', ...)`; (b) line 78 uses `new URL('.', import.meta.url).pathname` which returns `/C:/...` on Windows; (c) line 79 string-matches `'/.npm/_cacache/tmp/'` which never matches Windows backslash paths; (d) `execSync('npm root -g')` needs `npm.cmd` on Windows. |
| B4 | `src/services/retrieval/search/lexical-search.ts:148, 339` | `spawnSync('rg', ...)` without `shell:true`. On Windows, Node will not auto-resolve `rg.exe` via PATHEXT — the call fails with `ENOENT` even when ripgrep is installed. `search_code` and `search_hybrid` are dead on Windows. |

### HIGH — breaks features but not install

| # | File | Bug |
|---|------|-----|
| B5 | `scripts/hooks/localnest-pre-tool.cjs:85`, `scripts/hooks/localnest-post-tool.cjs:61` | `spawnSync('localnest', ...)` — needs `localnest.cmd` on Windows or `shell:true`. Hooks silently fail. |
| B6 | `scripts/runtime/install-localnest-skill.mjs:394` | `spawnSync('npm', ['install', ...])` — needs `npm.cmd` on Windows. ML dependency install is silently skipped → embeddings disabled. |
| B7 | `scripts/quality/quality-audit.mjs:9` | `spawnSync('npm', ['audit', ...])` — same issue (only affects local quality script, not user install). |
| B8 | `src/runtime/config.ts:272`, `src/cli/commands/selftest-checks.ts:326` | `spawnSync('rg', ['--version'], ...)` runtime detection — same as B4. The "has_ripgrep" status check always returns false on Windows. |

### MEDIUM — verification gap

| # | File | Bug |
|---|------|-----|
| B9 | `.github/workflows/quality.yml` | Only runs on `ubuntu-latest`. Add a Windows job (or matrix) so tsc/lint/test/check run on `windows-latest` and Windows breakage is caught at PR time. |
| B10 | (missing) `.gitattributes` | No line-ending normalization. Windows users with `autocrlf=true` may receive CRLF in shell scripts and `.cjs` files, occasionally breaking shebangs/parsers. |

## Tasks

### Task 1 — Fix `bin/_boot.cjs` symlink-name detection (B1)

**Files:** `bin/_boot.cjs`, `package.json` (bin map), 8 new tiny shims under `bin/`.

**Approach:**
- Convert `bin/_boot.cjs` from "symlink-name detection" to "exported runner function". It exports `runTarget(targetScriptName)` which does the tsx-resolve + spawn dance.
- Create 8 per-bin shim `.cjs` files (`bin/localnest.cjs`, `bin/localnest-mcp.cjs`, `bin/localnest-mcp-setup.cjs`, `bin/localnest-mcp-doctor.cjs`, `bin/localnest-mcp-upgrade.cjs`, `bin/localnest-mcp-install-skill.cjs`, `bin/localnest-mcp-task-context.cjs`, `bin/localnest-mcp-capture-outcome.cjs`). Each is 3 lines:
  ```js
  #!/usr/bin/env node
  require('./_boot.cjs').runTarget('localnest.js'); // (or whatever its target ESM is)
  ```
- Update `package.json` bin entries from `bin/_boot.cjs` to the per-bin shim.
- Keep the legacy direct invocation path (`node _boot.cjs <script>`) for back-compat, but it's no longer the primary mechanism.

**Verify:** `npm run check` (tsc + node --check on every bin/ shim). Manually `node bin/localnest.cjs --help` and confirm it forwards to the right ESM target.

### Task 2 — Fix `expandHome()` cross-platform (B2)

**Files:** `src/runtime/config.ts`.

**Approach:** Replace the `process.env.HOME` reference with `os.homedir()`, accept both `~/` and `~\` separators on Windows.

```ts
import os from 'node:os';
export function expandHome(inputPath: string): string {
  return inputPath.replace(/^~(?=$|[\\/])/, os.homedir());
}
```

**Verify:** Existing `test/config.test.js` exercises it — make sure tests still pass after switching.

### Task 3 — Fix `preinstall-cleanup.mjs` Windows paths (B3)

**Files:** `scripts/runtime/preinstall-cleanup.mjs`.

**Approach:**
- Replace `path.join('/', ...)` Unix root build with proper drive-aware reconstruction (use `path.resolve`).
- Replace `new URL('.', import.meta.url).pathname` with `fileURLToPath(import.meta.url)` (already used elsewhere in the codebase).
- Replace `selfDir.includes('/.npm/_cacache/tmp/')` with `selfDir.includes(path.join('.npm', '_cacache', 'tmp'))`.
- Replace `execSync('npm root -g')` with `execSync(process.platform === 'win32' ? 'npm.cmd root -g' : 'npm root -g')`.

**Verify:** `node --check scripts/runtime/preinstall-cleanup.mjs`. Does not run during normal dev — the gating `npm_config_global === 'true'` check keeps it dormant locally.

### Task 4 — Fix `spawnSync('rg', ...)` on Windows (B4 + B8)

**Files:** `src/services/retrieval/search/lexical-search.ts`, `src/runtime/config.ts`, `src/cli/commands/selftest-checks.ts`.

**Approach:** Add a small helper `getRipgrepBinary()` that returns `'rg.exe'` on Windows, `'rg'` elsewhere, OR pass `shell: true` to `spawnSync` so PATHEXT resolves the executable. The `shell: true` approach is simplest and consistent with the existing `npm.cmd` pattern.

Implementation: introduce a tiny helper in `src/runtime/platform.ts` (new, ~20 lines):

```ts
export const isWindows = process.platform === 'win32';
export const RG_BIN = isWindows ? 'rg.exe' : 'rg';
export function platformShim(linuxName: string, suffix = '.cmd'): string {
  return isWindows ? `${linuxName}${suffix}` : linuxName;
}
```

Use `RG_BIN` in the 3 files above. Prefer the suffix approach over `shell:true` because `shell:true` requires arg quoting which is risky for user-supplied search queries.

**Verify:** `npm run check`, plus the existing `lexical-search` tests still pass on Linux (the `rg`/`rg.exe` selection only affects the binary name, not the args).

### Task 5 — Fix `spawnSync('localnest', ...)` and `spawnSync('npm', ...)` on Windows (B5, B6, B7)

**Files:** `scripts/hooks/localnest-pre-tool.cjs`, `scripts/hooks/localnest-post-tool.cjs`, `scripts/runtime/install-localnest-skill.mjs`, `scripts/quality/quality-audit.mjs`.

**Approach:** In each file, define a top-of-file constant:
```js
const isWindows = process.platform === 'win32';
const NPM_BIN = isWindows ? 'npm.cmd' : 'npm';
const LOCALNEST_BIN = isWindows ? 'localnest.cmd' : 'localnest';
```
Then replace string literals.

**Verify:** `node --check` on all four files. Hooks are .cjs — already covered by current `node --check` chain. Manual smoke: pipe a fake hook payload through `localnest-pre-tool.cjs` on Linux to confirm refactor didn't break anything.

### Task 6 — Add Windows CI matrix (B9)

**Files:** `.github/workflows/quality.yml`.

**Approach:** Convert the single `quality` job to a matrix:
```yaml
strategy:
  fail-fast: false
  matrix:
    os: [ubuntu-latest, windows-latest]
runs-on: ${{ matrix.os }}
```

Some steps should be Linux-only (`Audit quality` because it spawns `npm`) — keep them gated by `if: matrix.os == 'ubuntu-latest'`. Or fix `quality-audit.mjs` first (Task 5) and run on both. Run on both because we want Windows to actually exercise everything.

**Verify:** YAML parses, `actions/setup-node` and `npm ci` work on Windows. The very first PR with this workflow will surface anything we missed.

### Task 7 — Add `.gitattributes` (B10)

**Files:** `.gitattributes` (new).

**Approach:** Force LF line endings for all source/script files so Windows users with `core.autocrlf=true` don't end up with CRLF in shebangs:

```
* text=auto eol=lf
*.cjs text eol=lf
*.mjs text eol=lf
*.js  text eol=lf
*.ts  text eol=lf
*.sh  text eol=lf
*.md  text
*.png binary
*.jpg binary
*.ico binary
```

**Verify:** `git check-attr -a bin/_boot.cjs` reports `eol: lf`.

### Task 8 — Bump version + CHANGELOG (always-last)

**Files:** `package.json`, `CHANGELOG.md`.

**Approach:** Bump `0.3.0-beta.3` → `0.3.0-beta.4`. Prepend a CHANGELOG entry summarising the Windows compat sweep.

**Verify:** `node -e "console.log(require('./package.json').version)"` prints `0.3.0-beta.4`. `npm run check` green.

## Out of scope

- Setting up Wine / a Windows VM locally — defer to the new GitHub Actions Windows matrix; that's the real verification.
- Rewriting `install-localnest-skill.mjs` clients matrix for Windows — most clients (Cursor, Windsurf, Claude Desktop) have Windows builds with the same config-file paths, but verifying each one is its own task.
- Bundling ripgrep — keeping ripgrep optional, just making the spawn calls platform-correct.
- node:sqlite Windows quirks (Node 22.6+ ships it on Windows; no fix needed today).
- "Auto-update docs per version" memory — defer to a follow-up doc sweep.
