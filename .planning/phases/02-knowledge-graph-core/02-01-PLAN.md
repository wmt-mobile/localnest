**Slug normalization helper (not exported):**
```javascript
function toSlug(name) {
  return cleanString(name, 200)
    .toLowerCase()
    .replace(/[^a-z0-9]+/g, '_')
    .replace(/^_|_$/g, '');
}
```

**Export 5 functions:**

1. `addEntity(adapter, { name, type, properties, memoryId })`
   - Validate: name is required (throw Error if empty after cleanString)
   - Generate id via toSlug(name). If empty slug, throw Error.
   - type defaults to 'concept', properties defaults to {}, memoryId defaults to null
   - SQL: INSERT OR IGNORE INTO kg_entities (id, name, entity_type, properties_json, memory_id, created_at, updated_at) VALUES (?, ?, ?, ?, ?, ?, ?)
   - Use nowIso() for created_at and updated_at
   - Use stableJson(properties) for properties_json
   - If INSERT was ignored (changes === 0), do an UPDATE to refresh updated_at:
     UPDATE kg_entities SET updated_at = ? WHERE id = ?
   - Return: { id, name, entity_type, properties_json, memory_id, created_at, updated_at, created: changes > 0 }

2. `getEntity(adapter, entityId)`
   - Validate: entityId required
   - Query kg_entities by id
   - If not found, return null
   - Also fetch outgoing triples: SELECT * FROM kg_triples WHERE subject_id = ? AND valid_to IS NULL
   - Also fetch incoming triples: SELECT * FROM kg_triples WHERE object_id = ? AND valid_to IS NULL
   - Return: { ...entity, properties: JSON.parse(entity.properties_json), outgoing: [...], incoming: [...] }

3. `addTriple(adapter, { subjectId, predicate, objectId, validFrom, validTo, confidence, sourceMemoryId, sourceType })`
   - Validate: subjectId, predicate, objectId all required (throw if empty)
   - Generate id: `triple_${crypto.randomUUID()}`
   - Defaults: validFrom = null, validTo = null, confidence = 1.0, sourceMemoryId = null, sourceType = 'manual'
   - Clamp confidence to 0.0-1.0 range
   - SQL: INSERT INTO kg_triples (id, subject_id, predicate, object_id, valid_from, valid_to, confidence, source_memory_id, source_type, created_at) VALUES (?, ?, ?, ?, ?, ?, ?, ?, ?, ?)
   - Return: { id, subject_id: subjectId, predicate, object_id: objectId, valid_from, valid_to, confidence, source_memory_id: sourceMemoryId, source_type: sourceType, created_at }

4. `invalidateTriple(adapter, tripleId, validTo)`
   - Validate: tripleId required
   - validTo defaults to nowIso()
   - SQL: UPDATE kg_triples SET valid_to = ? WHERE id = ? AND valid_to IS NULL
   - Return: { id: tripleId, valid_to: validTo, invalidated: result.changes > 0 }

5. `queryEntityRelationships(adapter, entityId, { direction, includeInvalid } = {})`
   - Validate: entityId required
   - direction can be 'outgoing', 'incoming', or 'both' (default: 'both')
   - includeInvalid defaults to false
   - Build SQL conditions:
     - If not includeInvalid: add "AND t.valid_to IS NULL"
     - If direction is 'outgoing': WHERE t.subject_id = ?
     - If direction is 'incoming': WHERE t.object_id = ?
     - If direction is 'both': use UNION of both queries
   - JOIN kg_entities for subject and object names:
     ```sql
     SELECT t.*, s.name AS subject_name, o.name AS object_name
       FROM kg_triples t
       JOIN kg_entities s ON s.id = t.subject_id
       JOIN kg_entities o ON o.id = t.object_id
      WHERE t.subject_id = ? AND t.valid_to IS NULL
      ORDER BY t.created_at DESC
     ```
   - Return: { entity_id: entityId, direction, count: results.length, triples: results }

Also export the toSlug function as `normalizeEntityId` for use by Plan 02:

```javascript
export { toSlug as normalizeEntityId };
```

Keep file under 200 lines. Use cleanString() for all string inputs. No classes — pure functions only.
