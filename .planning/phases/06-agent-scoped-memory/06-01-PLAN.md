# Phase 06 Plan 01: Agent scopes table, diary CRUD, and recall isolation

## Objective

Add agent_id column to memory_entries, create agent_diary table, build scopes.js with diary CRUD, and filter recall by agent scope.

## Tasks

### Task 1: Schema v8 — agent_id column and agent_diary table
type="auto"

Add schema version 8 migration:
- ALTER TABLE memory_entries ADD COLUMN agent_id TEXT NOT NULL DEFAULT ''
- CREATE TABLE agent_diary (id TEXT PRIMARY KEY, agent_id TEXT NOT NULL, content TEXT NOT NULL, topic TEXT NOT NULL DEFAULT '', created_at TEXT NOT NULL)
- Index on agent_diary(agent_id, created_at DESC)
- Index on memory_entries(agent_id)
- Update SCHEMA_VERSION to 8
- Update ensureSchema CREATE TABLE to include agent_id

Done when: schema.js has version 8 migration with both changes.

### Task 2: scopes.js — diary write/read and entries/recall agent_id support
type="auto"

- Create src/services/memory/scopes.js with writeDiaryEntry and readDiaryEntries functions
- Update entries.js storeEntry to accept and persist agent_id
- Update recall.js to filter by agent_id (show own + global where agent_id = '')
- Update utils.js deserializeEntry to include agent_id
- Wire scopes methods through store.js and service.js

Done when: All four AGNT requirements are satisfied.

## Verification

- Schema v8 migration creates agent_id column and agent_diary table
- scopes.js exports writeDiaryEntry and readDiaryEntries
- recall filters by agent scope
- store.js and service.js expose diary methods
