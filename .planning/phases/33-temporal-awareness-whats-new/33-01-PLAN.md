<objective>
Implement `localnest_whats_new(since)` -- a single MCP tool that returns a cross-session delta covering new memories, new KG triples, changed files, and recent commits since a given timestamp (TEMPO-01). The response includes a natural-language summary under 200 characters (TEMPO-02). The magic value `"last_session"` resolves to the most recent `memory_entries.created_at` for the current agent + project scope (TEMPO-03). The existing `localnest_kg_as_of` tool is validated to work correctly with Phase 26's auto-stamped `valid_from` (TEMPO-04).

Purpose: Give agents a single "catch me up" tool to understand what changed since their last session, eliminating the need to call multiple tools (memory_list, kg_stats, etc.) to reconstruct context.

Output: A new `whats-new.ts` module (~150-200 lines), type additions to `types.ts`, method additions to `store.ts` and `service.ts`, and one new MCP tool in `memory-workflow.ts`.
</objective>

    - Build summary via string template, no LLM
    - Format: "{N} memories, {M} triples, {F} files, {C} commits since {date}" -- truncated to 200 chars

    - Date in summary formatted as short form (YYYY-MM-DD HH:MM)

    TEMPO-03: last_session resolution
    - resolveLastSession(adapter, agentId?, projectPath?) function:
      ```sql
      SELECT MAX(created_at) AS last_ts
        FROM memory_entries
       WHERE (agent_id = ? OR ? IS NULL OR ? = '')
         AND (scope_project_path = ? OR ? IS NULL OR ? = '')
      ```
    - If no rows match, return '1970-01-01T00:00:00Z' (epoch) and set resolved_from = 'epoch'
    - If since is a valid ISO timestamp, use it directly and set resolved_from = 'timestamp'
    - If since is 'last_session', resolve and set resolved_from = 'last_session'

    Implementation details for whats-new.ts:
    - New memories query:
      ```sql
      SELECT id, title, kind, created_at FROM memory_entries
       WHERE created_at > ?
         AND (agent_id = ? OR ? IS NULL)
         AND (scope_project_path = ? OR ? IS NULL)
       ORDER BY created_at DESC
      ```
      - Count query: same WHERE but SELECT COUNT(*)
      - Items query: same WHERE but LIMIT ?

    - New triples query:
      ```sql
      SELECT t.id, s.name AS subject_name, t.predicate, o.name AS object_name, t.created_at
        FROM kg_triples t
        JOIN kg_entities s ON s.id = t.subject_id
        JOIN kg_entities o ON o.id = t.object_id
       WHERE t.created_at > ?
       ORDER BY t.created_at DESC
      ```
      - Count query: SELECT COUNT(*) FROM kg_triples WHERE created_at > ?
      - Items query: same with LIMIT

    - Files changed query:
      - Extract distinct file paths from links_json of memories created since the timestamp
      ```sql
      SELECT DISTINCT links_json FROM memory_entries
       WHERE created_at > ?
         AND links_json != '[]'
         AND (agent_id = ? OR ? IS NULL)
         AND (scope_project_path = ? OR ? IS NULL)
      ```
      - Parse each links_json, collect unique paths from link objects with non-empty path fields
      - Return count + array of distinct paths (capped at limit)

    - Recent commits query:
      ```sql
      SELECT id, title, event_type, created_at FROM memory_events
       WHERE created_at > ?
         AND event_type IN ('commit', 'pr_merge', 'deploy', 'release')
         AND (scope_project_path = ? OR ? IS NULL)
       ORDER BY created_at DESC
       LIMIT ?
      ```
      - Count query: same WHERE but SELECT COUNT(*)

    - Summary generation:
      - Parts array: push "{count} new memories" if count > 0, "{count} new triples" if count > 0, etc.
      - Join with ", " and append " since {shortDate}"
      - Truncate to 200 chars with ellipsis if needed
      - If all counts are 0: "No changes since {shortDate}"
