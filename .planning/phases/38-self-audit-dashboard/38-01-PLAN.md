**Status:** Executing

## Implementation Plan

### File 1: `src/services/memory/audit/dashboard.ts` (NEW)

Core audit logic. Single `runAudit(adapter)` function that executes SQL queries and assembles the dashboard.

**AUDIT-01 -- Memory Coverage by Project:**
- `SELECT scope_project_path, COUNT(*) FROM memory_entries GROUP BY scope_project_path`
- Report projects with/without memories and per-project counts

**AUDIT-02 -- KG Density Metrics:**
- Reuse entity/triple counts from simple COUNT queries
- Connected components via recursive CTE on kg_triples (walk from each entity, group by root)
- Orphaned entities: entities with zero triples (not in subject_id or object_id of any active triple)
- Duplicate triples: `(subject_id, predicate, object_id)` where `valid_to IS NULL` appearing more than once

**AUDIT-03 -- Unpopulated Nests & Broken Bridges:**
- Memories with empty nest OR empty branch: `WHERE nest = '' OR branch = ''`
- Broken bridges: triples where subject or object entity doesn't exist in kg_entities

**AUDIT-04 -- Stale Memories:**
- Never recalled: `last_recalled_at IS NULL`
- Low importance: `importance < 30`
- Older than 30 days: `created_at < datetime('now', '-30 days')`
- All three conditions combined = stale
