Add actor attribution to memory entries: schema column, write/read paths, filters, and agent_prime surface.

Purpose: Multi-agent systems need to know which actor (user, agent, tool) created a memory for filtering and attribution. actor_id is a peer of agent_id — agent_id is the scoping/visibility axis, actor_id is the attribution axis.
Output: actor_id column on memory_entries, propagated through all write paths (auto-inferred from agent_id when omitted), filterable on recall/list, and present on CompactMemory items in agent_prime.
</objective>
