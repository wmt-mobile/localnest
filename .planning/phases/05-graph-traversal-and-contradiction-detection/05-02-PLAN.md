Append to `src/services/memory/graph.js` a new exported async function `discoverBridges(adapter, { nest } = {})`.

Parameters:
- `nest` (optional) — if provided, only find bridges involving this nest. Clean with `cleanString(nest, 200)`.

The SQL query finds entities that appear in triples where the subject's nest differs from the object's nest. An entity's nest comes from its linked memory entry (kg_entities.memory_id -> memory_entries.nest). Also check triples where the entity's source_memory_id links to a memory with a nest.

The approach: find all currently-valid triples where the subject entity's associated nest differs from the object entity's associated nest. An entity's nest is determined by joining kg_entities -> memory_entries via memory_id.

SQL:
```sql
SELECT DISTINCT
  t.id AS triple_id,
  t.subject_id,
  s.name AS subject_name,
  t.predicate,
  t.object_id,
  o.name AS object_name,
  ms.nest AS subject_nest,
  mo.nest AS object_nest
FROM kg_triples t
JOIN kg_entities s ON s.id = t.subject_id
JOIN kg_entities o ON o.id = t.object_id
LEFT JOIN memory_entries ms ON ms.id = s.memory_id AND ms.nest != ''
LEFT JOIN memory_entries mo ON mo.id = o.memory_id AND mo.nest != ''
WHERE t.valid_to IS NULL
  AND ms.nest IS NOT NULL
  AND mo.nest IS NOT NULL
  AND ms.nest != mo.nest
ORDER BY ms.nest, mo.nest, s.name
```

If `nest` is provided, add `AND (ms.nest = ? OR mo.nest = ?)` with params `[nest, nest]`.

Also check bridges via source_memory_id on the triple itself for cases where entities lack direct memory_id links:
```sql
UNION
SELECT DISTINCT
  t.id AS triple_id,
  t.subject_id,
  s.name AS subject_name,
  t.predicate,
  t.object_id,
  o.name AS object_name,
  COALESCE(ms.nest, mt.nest) AS subject_nest,
  COALESCE(mo.nest, mt.nest) AS object_nest
FROM kg_triples t
JOIN kg_entities s ON s.id = t.subject_id
JOIN kg_entities o ON o.id = t.object_id
LEFT JOIN memory_entries ms ON ms.id = s.memory_id
LEFT JOIN memory_entries mo ON mo.id = o.memory_id
LEFT JOIN memory_entries mt ON mt.id = t.source_memory_id
WHERE t.valid_to IS NULL
  AND COALESCE(ms.nest, mt.nest, '') != ''
  AND COALESCE(mo.nest, mt.nest, '') != ''
  AND COALESCE(ms.nest, mt.nest) != COALESCE(mo.nest, mt.nest)
```

Actually — simplify. Use a single query that is clean and maintainable. The primary bridge detection path: entities linked to memory entries in different nests.

```sql
SELECT DISTINCT
  t.id AS triple_id,
  t.subject_id,
  s.name AS subject_name,
  t.predicate,
  t.object_id,
  o.name AS object_name,
  ms.nest AS subject_nest,
  mo.nest AS object_nest
FROM kg_triples t
JOIN kg_entities s ON s.id = t.subject_id
JOIN kg_entities o ON o.id = t.object_id
JOIN memory_entries ms ON ms.id = s.memory_id
JOIN memory_entries mo ON mo.id = o.memory_id
WHERE t.valid_to IS NULL

```

```javascript
     FROM kg_triples t
     JOIN kg_entities e ON e.id = t.object_id
    WHERE t.subject_id = ?
      AND t.predicate = ?
      AND t.object_id != ?
      AND t.valid_to IS NULL`,

```

Then, after the INSERT statement, include the contradictions in the return value. Change the return object to include a `contradictions` array:

```javascript
const contradictions = conflicting.map(c => ({
  existing_triple_id: c.id,
  existing_object_id: c.object_id,
  existing_object_name: c.object_name
}));

return {
  id,
  subject_id: subId,
  predicate: pred,
  object_id: objId,
  valid_from: vFrom,
  valid_to: vTo,
```
