<objective>
Add transaction safety and per-version tracking to the schema migration runner.

Purpose: The current runMigrations() sets schema_version once at the end of all migrations. A crash mid-migration leaves the database in an inconsistent state with no record of which versions succeeded. This plan wraps each version bump in its own SQLite transaction and stamps schema_version immediately after each succeeds, satisfying MIGR-01 and MIGR-02.

Output: Updated adapter.js with transaction support, updated schema.js with transactional per-version migration runner.
</objective>

  <action>
Add a `transaction(fn)` method to `NodeSqliteAdapter` that:

1. Calls `this.db.exec('BEGIN')` to start a transaction
2. Calls `await fn(this)` passing the adapter itself so the callback can use `this.exec()`, `this.run()`, `this.get()`, `this.all()`
3. On success: calls `this.db.exec('COMMIT')`
4. On error: calls `this.db.exec('ROLLBACK')` then re-throws the error

The method signature: `async transaction(fn)` where `fn` is `async (adapter) => { ... }`

Implementation (add after the `all` method, before the closing brace of the class):

```javascript
async transaction(fn) {
  this.db.exec('BEGIN');
  try {
    const result = await fn(this);
    this.db.exec('COMMIT');
    return result;
  } catch (err) {
    this.db.exec('ROLLBACK');
    throw err;
  }
}
```

NOTE: We use `this.db.exec()` directly (not `this.exec()`) because `this.exec()` is async-wrapped and we want the raw synchronous call for transaction control statements. Actually, since `this.exec()` just calls `this.db.exec()` synchronously and returns, either works -- but using `this.db.exec()` for BEGIN/COMMIT/ROLLBACK makes the intent clearer and avoids unnecessary promise wrapping for these critical control statements.

Do NOT modify any existing methods. This is purely additive -- per MIGR-01.

**Step 4: Run each migration in its own transaction with immediate version stamping**
```javascript
for (const { version, migrate } of migrations) {
  if (currentVersion >= version) {
    continue;  // MIGR-02: skip already-applied versions
  }
  await adapter.transaction(async (ad) => {
    await migrate(ad);
    // Stamp version INSIDE the transaction so it rolls back if migration fails
    await ad.run(
      "INSERT OR REPLACE INTO memory_meta(key, value) VALUES ('schema_version', ?)",
      [String(version)]
    );
  });
}
```
