    - Calls getFileMemoryHints(adapter, filePath) with suggest_update = true
    - Returns { file_path, hints: ProactiveHint[] } where each hint has suggest_update: true
    - Agent sees memory IDs that may need updating after the file was modified

    HOOK-09: Non-blocking hooks
    - All hint lookups are wrapped in try/catch in the MCP handler
    - If the hint query fails, the normal tool response is returned unmodified
    - localnest_file_changed failures return { file_path, hints: [], error: "..." }

    Implementation details:
    - SQLite query uses JSON_EACH to unnest links_json and match path. Query:
      ```sql
      SELECT DISTINCT me.id, me.title, me.importance, me.kind, me.summary
        FROM memory_entries me, JSON_EACH(me.links_json) AS je
       WHERE me.importance >= 70
         AND me.status = 'active'
         AND JSON_EXTRACT(je.value, '$.path') = ?
       ORDER BY me.importance DESC
       LIMIT 5
      ```
    - The query caps at 5 hints to avoid noisy responses
    - Path matching is exact (no glob, no substring) -- the link path must equal the requested file path
    - Path normalization: both the query path and the stored path should be compared after basic normalization (remove trailing slashes, resolve . and ..). Use Node.js path.resolve() on both sides before comparison. However, since stored paths may vary, we do exact match in SQL and also try a path.resolve'd version if the first query returns 0 rows.

    2) Create src/services/memory/proactive-hints.ts (~80 lines):
       ```typescript
       import path from 'node:path';
       import type { Adapter, ProactiveHint, ProactiveHintResult } from './types.js';

       const IMPORTANCE_THRESHOLD = 70;
       const MAX_HINTS = 5;
       const SUMMARY_EXCERPT_LENGTH = 120;

       interface HintRow {
         id: string;
         title: string;
         importance: number;
         kind: string;
         summary: string;
       }

       /**
        * Find memories with importance >= 70 that have a link matching
        * the given file path. Returns up to 5 hints, sorted by importance desc.
        *
        * Non-blocking: callers should wrap in try/catch.
        */
       export async function getFileMemoryHints(
         adapter: Adapter,
         filePath: string,
         suggestUpdate: boolean = false
       ): Promise<ProactiveHintResult> {
         const normalizedPath = path.resolve(filePath);

         // Try exact match first, then normalized path
         let rows = await queryHintsByPath(adapter, filePath);
         if (rows.length === 0 && normalizedPath !== filePath) {
           rows = await queryHintsByPath(adapter, normalizedPath);
         }

         const hints: ProactiveHint[] = rows.map(row => ({
           memory_id: row.id,
           title: row.title,
           importance: row.importance,
           kind: row.kind,
           summary_excerpt: row.summary.length > SUMMARY_EXCERPT_LENGTH
             ? row.summary.slice(0, SUMMARY_EXCERPT_LENGTH) + '...'
             : row.summary,
           suggest_update: suggestUpdate
         }));

         return { file_path: filePath, hints };
       }

       async function queryHintsByPath(
         adapter: Adapter,
         filePath: string
       ): Promise<HintRow[]> {
         return adapter.all<HintRow>(
           `SELECT DISTINCT me.id, me.title, me.importance, me.kind, me.summary
              FROM memory_entries me, JSON_EACH(me.links_json) AS je
             WHERE me.importance >= ?
               AND me.status = 'active'
               AND JSON_EXTRACT(je.value, '$.path') = ?
             ORDER BY me.importance DESC
             LIMIT ?`,
           [IMPORTANCE_THRESHOLD, filePath, MAX_HINTS]
         );
       }
       ```
