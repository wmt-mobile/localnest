# Phase 8 Plan 1: Conversation Ingestion - Schema, Parsers, Entity Extraction, and Dedup

## Objective

Create src/services/memory/ingest.js with Markdown and JSON conversation parsers, per-turn memory storage, rule-based entity extraction for KG triples, semantic dedup integration, and file re-ingestion tracking via conversation_sources table (schema v9).

## Context

- @src/services/memory/entries.js — storeEntry for creating memory entries
- @src/services/memory/kg.js — addTriple/ensureEntity for KG triples
- @src/services/memory/dedup.js — checkDuplicate for semantic dedup
- @src/services/memory/schema.js — schema migrations, currently at v8
- @src/services/memory/store.js — MemoryStore class to wire ingest methods
- @src/services/memory/service.js — MemoryService class to wire ingest methods

## Tasks

### Task 1: Schema v9 migration for conversation_sources table
type="auto"

Add conversation_sources table to track ingested files by path + SHA-256 hash.

**Schema:**
```sql
CREATE TABLE IF NOT EXISTS conversation_sources (
  id TEXT PRIMARY KEY,
  file_path TEXT NOT NULL,
  file_hash TEXT NOT NULL,
  turn_count INTEGER NOT NULL DEFAULT 0,
  memory_ids_json TEXT NOT NULL DEFAULT '[]',
  ingested_at TEXT NOT NULL
);
CREATE UNIQUE INDEX IF NOT EXISTS idx_conv_sources_path_hash ON conversation_sources(file_path, file_hash);
```

Update SCHEMA_VERSION to 9. Add migration entry.

**Done when:** Schema version is 9 with conversation_sources table in ensureSchema and migration.

### Task 2: Create ingest.js with Markdown parser, JSON parser, entity extraction, and dedup
type="auto"

Create src/services/memory/ingest.js with:

1. **parseMarkdownConversation(text)** — Split markdown into turns. Detect patterns like `## User`, `## Assistant`, `**User:**`, `> `, etc.
2. **parseJsonConversation(data)** — Accept array of {role, content, timestamp} objects.
3. **extractEntities(text)** — Rule-based regex heuristics: capitalized noun phrases, quoted terms, URLs, file paths, code identifiers.
4. **ingestMarkdown(adapter, embeddingService, opts)** — Full pipeline: parse, dedup per turn, store as memory entries, extract entities, create KG triples, track source file.
5. **ingestJson(adapter, embeddingService, opts)** — Same pipeline for JSON format.
6. **checkAlreadyIngested(adapter, filePath, hash)** — Check conversation_sources by path+hash.

**Done when:** All functions exported and handle edge cases (empty content, missing fields).

### Task 3: Wire ingest methods through store.js and service.js
type="auto"

Add ingestMarkdown and ingestJson methods to MemoryStore and MemoryService classes.

**Done when:** Both classes expose ingestMarkdown(opts) and ingestJson(opts).

## Verification

- ingest.js exists with all exported functions

## Success Criteria

- Markdown conversations parse into per-turn entries
- JSON conversations parse into per-turn entries

- Entity extraction creates KG triples via regex heuristics
- Dedup prevents duplicate turns
- Re-ingestion by hash is skipped
