```
# Auto-publish to npm AND create GitHub release when package.json version
# changes on main or release/* branches.
#
# SAFETY GUARDS (in execution order):
#  1. Only triggers on pushes to main/release/* (NOT on PRs, NOT on tags)
#  2. Only fires when package.json was modified in the push
#  3. Skips if version hasn't changed vs previous commit
#  4. Fails if version already exists on npm registry
#  5. Runs full quality pipeline before publishing
#  6. Verifies pack tarball is sane (file count + critical files)
#  7. Uses npm OIDC trusted publishing for provenance attestation
#  8. Auto-detects @beta/@alpha/@next/@latest tag from version string
#  9. Concurrency lock prevents overlapping publishes
# 10. Optional npm-publish environment for manual approval gate
# 11. GitHub release created ONLY after npm publish succeeds
#
# AUTHENTICATION:
#  Uses OIDC trusted publishing (no secrets required).
#  Configure at: https://www.npmjs.com/package/localnest-mcp/access
#  Fallback: Set NPM_TOKEN secret if OIDC is not yet configured.
```

**Name:** `Release`

**Triggers:**

**Job:** `release` with name `Publish to npm + GitHub Release`, runs-on `ubuntu-latest`, timeout-minutes 20, environment `npm-publish`

**Steps (in exact order):**

1. **Checkout** -- `actions/checkout@de0fac2e4500dabe0009e67214ff5f5447ce83dd` (v6) with `fetch-depth: 2`

2. **Setup Node** -- `actions/setup-node@6044e13b5dc448c55e2357c09f80417699197238` (v6) with node-version 22, cache npm, registry-url `https://registry.npmjs.org`

3. **Upgrade npm for OIDC support** -- `run: npm install -g npm@latest`

4. **Read local version** -- id `local`, run:
   ```
   echo "version=$(node -p "require('./package.json').version")" >> "$GITHUB_OUTPUT"
   ```

5. **Read previous version** -- id `previous`, run:
   ```
   PREV=$(git show HEAD~1:package.json 2>/dev/null | node -p "JSON.parse(require('fs').readFileSync(0, 'utf8')).version" 2>/dev/null || echo "none")
   echo "version=$PREV" >> "$GITHUB_OUTPUT"
   ```

6. **Check if version changed** -- id `changed`, run the exact same logic as existing release.yml comparing local vs previous, setting `should_release=true` or `should_release=false`

7. **Check npm registry for collision** -- if `steps.changed.outputs.should_release == 'true'`, query `npm view "$PKG@$VER" version` and fail if it already exists

8. **Install dependencies** -- if should_release, run `npm ci` (full install including postinstall, because tests may need ML deps)

9. **Run quality pipeline** -- if should_release, run `npm run quality`

10. **Verify pack tarball** -- if should_release, run `npm pack --dry-run`, check for 50+ files and required entries (`bin/localnest.js`, `bin/localnest-mcp.js`, `src/app/mcp-server.ts`, `package.json`)

11. **Determine npm tag** -- id `tag`, if should_release, detect tag from version string:
    - Contains `-beta` -> tag=beta, prerelease=true
    - Contains `-alpha` -> tag=alpha, prerelease=true
    - Contains `-rc` -> tag=next, prerelease=true
    - Otherwise -> tag=latest, prerelease=false

12. **Publish to npm (OIDC)** -- if should_release, run:
    ```yaml
    - name: Publish to npm
      if: steps.changed.outputs.should_release == 'true'
      run: npm publish --provenance --access public --tag ${{ steps.tag.outputs.tag }}
    ```
    CRITICAL: Do NOT set `NODE_AUTH_TOKEN` env var. OIDC handles auth via the `id-token: write` permission. Setting NODE_AUTH_TOKEN (even empty) causes npm to skip OIDC and attempt token auth.

    If a fallback is needed for initial setup, add a second command after `||`:
    ```
    npm publish --provenance --access public --tag $TAG || npm publish --access public --tag $TAG
    ```
    But do NOT add `env: NODE_AUTH_TOKEN: ${{ secrets.NPM_TOKEN }}` -- if the user needs token fallback, they should set it as a repo-level env var on the npm-publish environment, not on this step.

13. **Create GitHub Release** -- if should_release, use `gh release create` with:
    - Tag: `v${{ steps.local.outputs.version }}`
    - Target: `${{ github.ref_name }}`
    - `--prerelease` flag if `steps.tag.outputs.prerelease == 'true'`
    - `--generate-notes` for auto-generated release notes
    - Skip if release already exists (idempotent)
    - `env: GH_TOKEN: ${{ secrets.GITHUB_TOKEN }}`

14. **Summary** -- if should_release, write to `$GITHUB_STEP_SUMMARY` with package name, version, previous version, npm tag, GitHub release tag, and provenance status
