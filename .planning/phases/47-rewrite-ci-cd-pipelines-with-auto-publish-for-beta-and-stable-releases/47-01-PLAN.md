**Concurrency:**
```yaml
concurrency:
  group: quality-${{ github.workflow }}-${{ github.ref }}
  cancel-in-progress: ${{ github.event_name == 'pull_request' }}
```

**Job:** `quality` with name `Quality Pipeline`, runs-on `ubuntu-latest`, timeout-minutes 15

**Steps (in order):**
1. Checkout using `actions/checkout@de0fac2e4500dabe0009e67214ff5f5447ce83dd` (v6)
2. Setup Node using `actions/setup-node@6044e13b5dc448c55e2357c09f80417699197238` (v6) with node-version 22 and cache npm (NOTE: this is the NEWER SHA from release.yml, replacing the old one `53b83947a5a98c8d113130e565377fae1a50d02f`)
3. Install (skip postinstall): `npm ci --ignore-scripts`
4. Typecheck: `npm run check`
5. Lint: `npm run lint`
6. Install ML dependencies for tests: `node scripts/runtime/install-localnest-skill.mjs --quiet 2>/dev/null || true`
7. Test with coverage: `npm run test:coverage`
8. Check circular deps: `npm run quality:cycles`
9. Check unused deps: `npm run quality:deps`
10. Package quality: `npm run quality:package`
11. Audit quality: `npm run quality:audit`

IMPORTANT: Do NOT use `npm run quality` as a single command because it runs `npm run check` which includes `tsc --noEmit` that needs the full install. Instead, break the quality pipeline into individual steps so that steps 4-5 (lint/typecheck) run after `--ignore-scripts` install, and steps 7-11 (test + quality checks) run after ML deps are installed. The `check` script (tsc + node --check) works fine without the postinstall because it only needs TypeScript and the declared dependencies in node_modules, not the ML runtime.

Do NOT set NODE_AUTH_TOKEN or any npm secrets -- this workflow never publishes.
