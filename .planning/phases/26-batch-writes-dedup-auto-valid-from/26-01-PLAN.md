<objective>
Add transactional batch write tools for KG entities and triples (BATCH-01, BATCH-02), with write-time triple dedup on (subject_id, predicate, object_id) where valid_to IS NULL (BATCH-05), all-or-nothing rollback on unexpected failures with per-item validation errors collected into the response (BATCH-04), and auto-stamp valid_from = NOW() at the shared insert site so both single and batch writes get correct temporal validity (BATCH-06).

Purpose: Lay the foundation for v0.2.0 — every later phase (terse responses, contradiction fix, KG fusion, agent_prime, find, whats_new) depends on the ability to write KG data in bulk without per-row overhead, with dedup catching repeated facts and valid_from auto-populated for kg_as_of queries.

Output: Two new MCP tools (`localnest_kg_add_entities_batch`, `localnest_kg_add_triples_batch`), a new `kg-batch.ts` module under 500 lines, a one-line behavior change to existing `addTriple` for valid_from, store/service plumbing, and a test file covering happy path, dedup, rollback, oversized batch, and the valid_from auto-stamp regression.
</objective>
