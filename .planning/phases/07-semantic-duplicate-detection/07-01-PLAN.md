# Phase 07 Plan 01: Embedding Similarity Gate and Integration

## Objective
Create dedup.js with checkDuplicate() that compares embedding cosine similarity against existing entries, then wire it into entries.js storeEntry and event-capture.js promotion flow.

## Context
- @src/services/retrieval/core/relevance.js -- cosineSimilarity function
- @src/services/retrieval/embedding/service.js -- EmbeddingService.embed()
- @src/services/memory/entries.js -- storeEntry to wire dedup before insert
- @src/services/memory/event-capture.js -- captureEvent promotion to wire dedup before promotion
- @src/services/memory/store.js -- MemoryStore class to add checkDuplicate method
- @src/services/memory/service.js -- MemoryService class to add checkDuplicate method

## Tasks

### Task 1: Create dedup.js with checkDuplicate function
type="auto"

Create src/services/memory/dedup.js:
- Export checkDuplicate(adapter, embeddingService, content, { threshold, nest, branch, projectPath })
- Default threshold: 0.92
- Embed the incoming content via embeddingService.embed()
- Query recent memory_entries that have embedding_json (limit 200, same scope filters)
- Compare cosine similarity using cosineSimilarity from retrieval/core/relevance.js
- Return { isDuplicate: true, match: { id, title, similarity } } when similarity >= threshold
- Return { isDuplicate: false } when no match found
- Handle edge cases: embeddings disabled, empty content, no existing entries

**Done when:** dedup.js exists with checkDuplicate exported and handles all edge cases.

### Task 2: Wire dedup into entries.js, event-capture.js, store.js, service.js
type="auto"

- entries.js storeEntry: after fingerprint check, before INSERT, call checkDuplicate. If duplicate found, return { created: false, duplicate: true, memory: matchingEntry }.
- event-capture.js captureEvent: before promotion (storeEntry call), run checkDuplicate. If duplicate, set status='duplicate' and skip promotion.
- store.js MemoryStore: add checkDuplicate(content, opts) method that delegates to dedup.js
- service.js MemoryService: add checkDuplicate(content, opts) method that delegates to store

**Done when:** storeEntry rejects semantic duplicates, event promotion skips duplicates, and checkDuplicate is callable from service/store.

## Verification
- dedup.js exports checkDuplicate
- entries.js imports and calls checkDuplicate before insert
- event-capture.js calls checkDuplicate before promotion
- store.js and service.js expose checkDuplicate

## Success Criteria
- DEDUP-01: Configurable threshold (default 0.92) checked before storage
- DEDUP-02: checkDuplicate callable with content and threshold
- DEDUP-03: Returns matching entry details on duplicate
- DEDUP-04: Event capture runs dedup before promotion
